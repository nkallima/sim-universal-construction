Detailed Description

This file exposes the API of the Osci combining object. An example of use of this API is provided in benchmarks/oscibench.c file.

Author
Nikolaos D. Kallimanis (nkall.nosp@m.ima@.nosp@m.gmail.nosp@m..com)

For a more detailed description see the original publication: Panagiota Fatourou, and Nikolaos D. Kallimanis. "Lock Oscillation: Boosting the Performance of Concurrent Data Structures. In proceedings of the 21st International Conference on Principles of Distributed Systems (OPODIS 2017), 2018.

◆ OsciFiberRec

struct OsciFiberRec

OsciFiberRec stores the data of an announced request by a fiber.

Data Fields
volatile ArgVal arg_ret This variable stores the argument of the request and the return value after the request is applied.
volatile int32_t pid The pid of the fiber that announced this request.
volatile int16_t locked Whenever it is equal to false, the fiber is the combiner; otherwise the fiber waits until a combiner apply its request.
volatile int16_t completed If true, the request is applied and the fiber returns its return value.

◆ OsciNode

struct OsciNode

OsciNode is used as a combining point for the fibers of the current posix thread.

Data Fields
volatile struct OsciNode * next next point to the next request announced by some fiber of the current posix thread.
volatile int32_t toggle toggle bit for the current combining point.
volatile int32_t door door may be equal with any of the following: _OSCI_DOOR_INIT or _OSCI_DOOR_OPENED or _OSCI_DOOR_LOCKED.
volatile OsciFiberRec * rec points to an announced request.

◆ OsciThreadState

struct OsciThreadState

OsciThreadState stores each posix thread's local state for a single instance of Osci. For each instance of Osci, a different instance of OsciThreadState should be used.

Data Fields
volatile OsciNode next_node[2] An array of two empty request structs that will be used for announcing future requests.
int toggle A toggle-bit used by the Osci object.

◆ OsciStruct

struct OsciStruct

OsciStruct stores the state of an instance of the an Osci combining object. OsciStruct should be initialized using the OSciInit function.

Data Fields
uint32_t fibers_per_thread The number of fibers per posix thread that the current instance of Osci supports (this is set by using the OsciInit functionality).
uint32_t groups_of_fibers The number of posix threads that are accessing the current instance of Osci. Each posix thread contains fibers_per_thread fibers.
ptr_aligned_t * current_node Combining points for the current instance of Osci.

void OsciInit ( OsciStruct *  l,
uint32_t  nthreads,
uint32_t  fibers_per_thread 
)

This function initializes an instance of the Osci combining object.

This function should be called once (by a single thread) before any other thread tries to apply any request by using the OsciApplyOp function.

Parameters
lA pointer to an instance of the Osci combining object.
nthreadsThe number of threads that will use the Osci combining object.
fibers_per_threadThe number of fibers per posix thread that the current instance of Osci supports (this is set by using the OsciInit functionality).
